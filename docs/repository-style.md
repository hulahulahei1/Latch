# Repository Style

LATCH should look like a protocol repository, not a token landing page.

---

## Tone

Use:

```text
verifiable execution
agent-native
Base-native
proof-gated
hidden computation
capsule
round
credential
unlock
```

Avoid:

```text
meme
pump
moon
tax token
profit promise
guaranteed buyback
APY
```

---

## README visual style

The README should be clean and technical:

- short headline;
- one strong tagline;
- simple mermaid diagram;
- concise module table;
- clear status / not production-ready section;
- security model visible above the fold;
- no excessive emojis;
- no trading language.

Recommended top block:

```md
# LATCH

Verifiable execution for AI agents on Base.

> Trust the proof, not the agent.
```

---

## Design language if a UI is added later

If a UI is ever added, keep it minimal:

- black / off-white / muted green;
- terminal-like panels;
- capsule hashes as primary visual objects;
- no cartoon AI;
- no casino colors;
- no price chart as the hero;
- show proof state, round state, and capsule state first.

The protocol should feel like infrastructure.
